# Skill: FI Posting Patterns (Option B — Direct FI BAPI)

## Purpose
Ensure all FI-GL documents generated by the Z addon are correctly structured, traceable, and compliant with SAP FI design patterns. All posting is direct via FI BAPIs — no RE-FX accounting engine.

## When to Use
- When designing any Z posting logic in CD-04
- When specifying FI document structures for UAT
- When defining account determination logic (ZRIF16_GL_MAP)
- When designing the simulation/preview mode
- When the accounting-traceability-check hook fires

## Mandatory Inputs
1. Posting type (e.g., "initial recognition", "monthly interest", "termination")
2. Confirmed FI BAPI name [TO BE CONFIRMED at T0-04]
3. Account determination table structure (ZRIF16_GL_MAP)
4. Parallel ledger requirement (confirmed or not)

## Expected Outputs
1. FI document structure (header + line items)
2. Account determination logic
3. Reference field strategy (Z IDs in XBLNR/BKTXT/ZUONR)
4. Simulation output format
5. Error handling specification
6. Reversal specification
7. ZRIF16_POST_LOG entry structure

## Posting Pattern Catalog

### Pattern 1: Initial Recognition
- **Trigger:** Lease accountant approves initial valuation and initiates posting
- **DR:** Right-of-Use Asset (account via ZRIF16_GL_MAP, posting type = INIT_ROU)
- **CR:** Lease Liability (account via ZRIF16_GL_MAP, posting type = INIT_LIAB)
- **Amount:** Lease liability at commencement (from ZRIF16_CALC_ITEM)
- **Cost object:** Cost center + profit center from Z contract master
- **Reference:** XBLNR = Z Contract ID, BKTXT = "IFRS16 INIT [Contract ID]"

### Pattern 2: Monthly Interest Accrual
- **Trigger:** Period-end batch job
- **DR:** Interest Expense (account via ZRIF16_GL_MAP, posting type = INT_EXP)
- **CR:** Lease Liability (account via ZRIF16_GL_MAP, posting type = INT_LIAB)
- **Amount:** Period interest from ZRIF16_SCHEDULE row
- **Reference:** XBLNR = Z Contract ID + Period

### Pattern 3: ROU Amortization
- **Trigger:** Period-end batch (via FI-AA depreciation run or direct FI entry — [TO BE CONFIRMED])
- **DR:** Amortization Expense (account via ZRIF16_GL_MAP, posting type = AMORT_EXP)
- **CR:** Accumulated Depreciation / ROU Asset (account via ZRIF16_GL_MAP or FI-AA)
- **Amount:** Period amortization from ZRIF16_SCHEDULE row

### Pattern 4: Lease Payment
- **Trigger:** Cash payment event or AP matching
- **DR:** Lease Liability (account via ZRIF16_GL_MAP, posting type = PYMT_LIAB)
- **CR:** AP clearing or bank (standard SAP account)
- **Amount:** Payment amount from ZRIF16_PYMT_SCHED

### Pattern 5: Termination / Derecognition
- **Trigger:** Contract termination event (CD-06 TERMINATED_EARLY)
- **DR:** Lease Liability (remaining balance)
- **CR:** ROU Asset (net book value)
- **DR/CR:** Gain or Loss on Termination (balancing entry)

### Pattern 6: Reclassification (Current/Non-Current)
- **Trigger:** CD-08 Reclassification Engine run
- **DR:** Long-term Lease Liability
- **CR:** Short-term Lease Liability
- **Amount:** Current portion from ZRIF16_RECLS_ITEM

## Traceability Requirements (OB-07)
Every FI document posted by the addon MUST:
- Carry Z Contract ID in at least one reference field
- Carry Z Calculation Run ID in at least one reference field
- Have a corresponding row in ZRIF16_POST_LOG with FI document reference (company code + year + doc number)

## Anti-Patterns
- ❌ Posting without simulation mode available first
- ❌ Hardcoding GL account numbers in ABAP (always use ZRIF16_GL_MAP)
- ❌ Allowing a batch to silently skip contracts with errors
- ❌ Creating FI documents without the Z contract ID reference
- ❌ Using RE-FX accounting engine for any posting
